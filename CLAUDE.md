# H&E + Multiplex Feature Generator

A pipeline for generating spatially-resolved, multi-channel feature maps from co-registered H&E and multiplex immunofluorescence (mIF) whole-slide images (Lin 2021 CRC Atlas / CRC02 tissue sample).

## Goal

Given a pair of H&E and multiplex OME-TIFF images, this pipeline:

1. Extracts tissue patches and aligns multiplex channel arrays to each patch
2. Runs GPU-accelerated cell segmentation (CellViT) to detect and classify cells
3. Refines cell type assignments using multiplex marker intensities
4. Produces per-patch feature layers (cell type maps, vasculature, signaling channels) suitable for downstream spatial analysis or model training

## Tech Stack

- **Language:** Python 3.13
- **Backend:** FastAPI + Uvicorn
- **Frontend:** Vanilla HTML/JS + OpenSeadragon 4.1.0 (Deep Zoom)
- **Key libs:** tifffile, Pillow, pandas, numpy, zarr, scipy
- **Testing:** pytest

## Pipeline Overview

```
Local machine                          Google Colab (GPU)
─────────────────────────────          ──────────────────────────────────
Stage 1: python -m stages.patchify
  → processed/he/*.png
  → processed/multiplex/*.npy
  → processed/index.json
         │
         │ upload patches (aws s3 sync)
         ▼
      [AWS S3]                    ──►   cellvit_colab_stage2.ipynb
                                          ↓
                                   processed/cellvit/*.json
         ◄─────────────────────────────────
         │ download results
         ▼
Stage 3: python -m stages.assign_cells
Stage 4: python -m stages.multiplex_layers
```


## Dev Commands

```bash
# Setup
conda create -n he-multiplex python=3.13
conda activate he-multiplex
pip install -r requirements.txt

# Stage 1 — Extract patches (ECC registration on by default)
python -m stages.patchify \
  --he-image data/CRC02-HE.ome.tif \
  --multiplex-image data/CRC02.ome.tif \
  --metadata-csv "data/CRC202105 HTAN channel metadata.csv" \
  --out processed/ \
  --patch-size 256 \
  --stride 256 \
  --tissue-min 0.1 \
  --channels CD31 Ki67 CD45 PCNA \
  --overview-downsample 64 \
  --vis-channels 0 10 20
  # --no-register   # disable ECC, fall back to mpp-scale only

# QC — verify H&E↔multiplex alignment (interactive or headless)
python -m tools.debug_match_he_mul \
  --he-image data/CRC02-HE.ome.tif \
  --multiplex-image data/CRC02.ome.tif \
  --metadata-csv "data/CRC202105 HTAN channel metadata.csv" \
  --downsample 64 \
  --index-json processed/index.json \
  --save-png /tmp/vis_registered.png

# Stage 2 — Upload to S3, run Colab notebook, download results
aws s3 sync processed/he/ s3://YOUR-BUCKET/he-feature-visualizer/processed/he/ --storage-class STANDARD_IA
# ... run notebooks/cellvit_colab_stage2.ipynb on Colab (T4 GPU) ...
aws s3 sync s3://YOUR-BUCKET/he-feature-visualizer/processed/cellvit/ processed/cellvit/

# Stage 3 — Assign cell types
python -m stages.assign_cells \
  --cellvit-dir processed/cellvit/ \
  --features-csv data/CRC02.csv \
  --out processed/

# Stage 4 — Multiplex layers
python -m stages.multiplex_layers \
  --multiplex-dir processed/multiplex/ \
  --metadata-csv "data/CRC202105 HTAN channel metadata.csv" \
  --out processed/

# WSI Viewer — Preprocess (one-time, 5–15 min) then serve
python preprocess.py --features data/crc02-features.csv --image data/CRC02-HE.ome.tif --out cache/
python server.py --image data/CRC02-HE.ome.tif --cache cache/ --port 8000
# Open http://127.0.0.1:8000

# Tests
pytest
```

## Key Architectural Notes

- **4-stage pipeline:** patchify locally → CellViT on Colab GPU → assign cells → multiplex layers
- **AWS S3 bridge:** patches uploaded to S3 for Colab access; results downloaded after inference
- **Colab resumable:** `SKIP_EXISTING=True` — safe to interrupt and resume without reprocessing
- **WSI viewer pipeline:** preprocess once, serve many — never re-read raw TIFF at query time
- **DZI tiling:** multi-level tile pyramid for efficient pan/zoom
- **OME-TIFF + zarr backend:** fast region access for multi-channel images
- **Parquet cell database:** indexed features for fast spatial queries
- **ECC affine registration (Stage 1):** `cv2.findTransformECC(MOTION_AFFINE)` on binary tissue masks (H&E HSV-Otsu vs MX DNA/DAPI ch0-Otsu) at overview resolution. Produces a 2×3 `warp_matrix` (H&E full-res → MX full-res) stored in `index.json`. Corrects translational/rotational misalignment between image pairs. Falls back to mpp-ratio scale if ECC fails. Disable with `--no-register`.
- **Alignment QC:** `python -m tools.debug_match_he_mul --index-json processed/index.json --save-png PATH` applies `warp_matrix` via `cv2.warpAffine` (with `WARP_INVERSE_MAP`) to visualize registered overlay. Use `--save-png` for headless verification without a display server.
