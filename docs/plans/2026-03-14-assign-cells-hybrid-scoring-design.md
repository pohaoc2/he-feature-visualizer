# Design: Assign Cells Hybrid Scoring (MX-first) for CRC

**Date:** 2026-03-14  
**Status:** APPROVED (user-reviewed in chat)  
**Primary files:** `stages/assign_cells.py`, `stages/extract_cell_features.py`, `tests/test_assign_cells.py`

## Motivation

The current Stage 3 rules are threshold-first and tuned for a broader historical label set.  
For current CRC usage, we want a stricter, clearer ontology:

1. **Cell type**: `cancer`, `immune`, `healthy`
2. **Cell state**: `proliferative`, `quiescent`, `dead`

We also want MX marker evidence to be primary while still using CellViT morphology as a prior.

## Goals

- Replace hard type assignment with **hybrid class scoring**
- Keep **MX-first** behavior with explicit CellViT priors
- Keep `dead` state as CellViT override
- Support dual data source flow:
  - precomputed features CSV
  - auto-extracted CellViT+MX features CSV
- Emit transparent conflict and confidence metadata

## Label Ontology

### Cell type (3 classes)

- `cancer`
- `immune`
- `healthy`

### Cell state (3 classes)

- `dead`
- `proliferative`
- `quiescent`

## CellViT Class Mapping

```python
CELL_TYPE_NAMES = {
    0: "background",    # ignore
    1: "Neoplastic",    # cancer prior
    2: "Inflammatory",  # immune prior
    3: "Connective",    # healthy prior
    4: "Dead",          # dead state override
    5: "Epithelial",    # split prior: cancer + healthy
}
```

## Marker Normalization

Use **per-slide percentile normalization** for each marker:

- For marker `m`, compute normalized value `x_m in [0, 1]` from its percentile rank in the full features table.
- Missing marker columns are allowed; formulas renormalize weights to available terms.

Rationale:
- robust to slide-level intensity shifts
- stable across marker dynamic ranges
- preserves monotonic ordering for scoring

## Hybrid Scoring Formulas (Locked v1)

### Panels

- `immune_t_panel = mean(CD3e, CD4, CD8a, CD45RO, FOXP3, PD-1)` (available-only mean)
- `immune_mb_panel = mean(CD20, CD68, CD163)` (available-only mean)
- `stromal_panel = mean(SMA, CD31, Desmin, Collagen)` (available-only mean)

Aliases are resolved (examples): `Pan-CK -> PanCK`, `E-cadherin -> Ecadherin`, `PD-1 -> PD1`.

### MX class scores

```text
S_MX(immune) = 0.55*CD45 + 0.30*immune_t_panel + 0.15*immune_mb_panel

S_MX(cancer) = 0.55*PanCK + 0.25*Ecad + 0.20*(1 - CD45)

S_MX(healthy) = 0.60*stromal_panel + 0.40*mean(1-PanCK, 1-CD45)
```

### CellViT priors

- type 1 (`Neoplastic`)   -> `{cancer:1.0, immune:0.0, healthy:0.0}`
- type 2 (`Inflammatory`) -> `{cancer:0.0, immune:1.0, healthy:0.0}`
- type 3 (`Connective`)   -> `{cancer:0.0, immune:0.0, healthy:1.0}`
- type 5 (`Epithelial`)   -> `{cancer:0.5, immune:0.0, healthy:0.5}`  (**split prior**)
- type 0 (`background`)   -> ignored cell
- type 4 (`Dead`)         -> state override path; type prior can be neutral

### Fused MX-first score

```text
S_fused(class) = 0.75 * S_MX(class) + 0.25 * S_CV(class)
type_final = argmax_class S_fused(class)
```

## State Rules (Locked)

1. `dead` if CellViT type is `4` (hard override)
2. else `proliferative` if `Ki67_norm >= 0.75` (p75)
3. else `quiescent`

## Mismatch, Confidence, and Explainability

### Mismatch definition

- `mx_top = argmax(S_MX)`
- `cv_top = argmax(S_CV)`
- mismatch if `mx_top != cv_top`

### Confidence

Let `margin = top1(S_fused) - top2(S_fused)`:

- `high`   if `margin >= 0.25`
- `medium` if `0.12 <= margin < 0.25`
- `low`    if `margin < 0.12`

If mismatch is true, downgrade one level (high->medium, medium->low).

### Per-cell output additions

- `cell_type` (`cancer|immune|healthy`)
- `cell_state` (`dead|proliferative|quiescent`)
- `cell_type_confidence` (`high|medium|low`)
- `mx_top_type`
- `cellvit_prior_type`
- `is_mismatch` (bool)
- optional score audit fields: `score_mx_*`, `score_fused_*`

## Data Flow

1. Load features table (provided CSV or auto-generated by `extract_cell_features`)
2. Normalize markers per slide
3. Build KDTree on coordinates
4. Match each CellViT cell to nearest feature row
5. Compute MX score, CellViT prior, fused score, final type
6. Compute state with dead override + Ki67 p75
7. Rasterize cell type/state overlays
8. Write summary metrics

## Error Handling and Data Quality

- Missing marker channels do not fail assignment; term weights are renormalized.
- If a whole score branch has no available markers, confidence is capped at `low`.
- Summary must include:
  - available vs missing channels
  - mismatch rate
  - confidence distribution
  - type/state distributions

## Out of Scope (for this redesign)

- Subtyping immune or healthy populations
- Training ML classifiers
- Changing Stage 2 CellViT inference outputs

## Compatibility Notes

- Legacy output folders remain unchanged (`cell_types/`, `cell_states/`, `cell_summary.json`).
- Existing dual mode remains:
  - use `--features-csv`, or
  - auto-generate CSV from `--cellvit-dir + --multiplex-dir`.
